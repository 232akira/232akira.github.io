主要作品集会基于以下两个方向：

🛠 方向1：引擎开发（Engine Development）
目标岗位：游戏引擎开发、游戏优化、底层系统开发

1️⃣重点

  • 优化 & 渲染管线定制
  • 自定义渲染器 / 后处理
  • 引擎扩展

2️⃣视频展示思路

  • 项目标题 + 目标（比如 “基于 Unity SRP 自定义渲染管线”）
  • 功能展示（比如 自定义的 Forward+ 渲染，或 Unity / Unreal 渲染器优化）
  • 代码 / 技术解析（可以穿插代码片段，比如如何实现优化）
  • 性能对比（前后效果对比，FPS对比）
  • 总结技术要点（如 “使用 Compute Shader 实现 Tile-based Culling 提升性能”）

3️⃣推荐作品

  • 基于 Unity / Unreal 实现一个小型渲染框架（比如基于URP / HDRP改造）
  • 做一个渲染优化 Demo（比如 Shadow Cascading / Foveated Rendering）
  • 实现实时 GI / 体积雾 / 后处理效果

🎨 方向2：图形开发（Graphics Programming）
目标岗位：图形开发工程师、Shader 开发、图形优化

1️⃣重点

  • 高质量 Shader 开发（HLSL / GLSL）
  • 光照模型 / PBR / 体积渲染
  • 计算着色器 / GPGPU

2️⃣视频展示思路

  • 作品名称 + 目标（如 “基于 GLSL 实现 PBR 渲染”）
  • Shader 效果展示（可以从多个角度转动模型，突出材质变化）
  • 代码 & 技术解析（穿插代码，讲解 Shader 关键部分）
  • 效果对比（基础 Blinn-Phong vs. PBR）

3️⃣推荐作品

  • 自定义 PBR Shader（展示不同的 Roughness、Metallic、AO 效果）
  • 水面 / 火焰 / 体积光 Shader（使用屏幕空间或计算着色器实现）
  • 基于 Shader Toy / Unity 实现屏幕后处理（如 Bloom, Motion Blur, SSAO）

GPU渲染管线

基本结构：

【模型数据】 -> 【顶点Shader】 -> 【图元装配及光栅化】 -> 【片元Shader】 -> 【输出合并】
            |-----------------------------GPU渲染管线-----------------------------|

1️⃣顶点Shader：

  • 将模型数据的模型空间坐标转换到对应的裁剪空间，即输出在裁剪空间下的顶点坐标
  • 【模型空间】 -> 世界空间 -> 相机空间 -> 【裁剪空间】，中间经过三个矩阵（Model,View,Projection）操作

2️⃣图元装配及光栅化

  • 硬件阶段
  • 生成片元并进行光栅化插值

3️⃣片段Shader

  • 每个片元调用片元shader给自身着色
  • 计算对应的颜色后输出

4️⃣输出合并

  • 输出到对应的帧缓冲区

一个完整的Shader通常由顶点Shader和片段Shader共同组成。
