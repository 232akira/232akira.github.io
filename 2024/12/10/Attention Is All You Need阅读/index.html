# Attention Is All You Need 阅读

先把作业贴过来，后面会慢慢补充。

《Attention Is All You Need》阅读笔记
2252805 - 康怡晨 - 24/12/10

这篇论文首先上来给人的感觉是题目特别短。因为之前看过的一些论文标题少说一两行，所以我以为这篇内容会很水。
结果完全完全相反，这里主要讲讲学到的自注意力机制。

# 先上个 github 链接，都去 star

# 首先，标题：Attention->self-attention，也就是这篇文章所研究的 Transformer 模型中最关键的自注意力机制。自注意力机制简单来说就是：让模型学会区分一句话中不同词语的重要程度。
正如 Abstract 部分所讲，Transformer 模型完全不需要递归和卷积，并且效果好、泛化能力也强，同时具有并行性。（相比像 RNN 这种怎么改都没法打破串行处理带来的限制的模型，优势很明显）
其中算法公式看不太懂，这里就先就不赘述了。
自注意力机制能够让 AI 更像人的思维一样去理解语言的上下文。当下流行的 openai 家的 GPT，全称叫 Generative Pre-trained Transformer，就是基与 Trans
-former 架构的自然语言处理模型。
回到论文，Introduction 提到，Attention 在输入输出序列建模过程其实并不少见（比如 RNN 的一部分就用到了）。这里注意到，自注意力机制并非本论文作者提出，而是已经有很多成功应用，但是完全依赖它的 Transformer 还是第一次。
文章中 3.2 提到了组成自注意力机制的几个向量：query 查询、key 键、value 值和 output 输出。注意力函数可被描述成：将 Query 和一系列 key-value 对映射到 output 的函数。实际中，query 打包成矩阵 Q，同理 key->K，value->V，得到输出矩阵的计算公式（详见 3.1.2）。
想起来上上次老师代课那节课，学了个产生词向量的 word2vec 模型，与它相比看，transformer 能通过上下文来动态计算向量，显然更优。
看不完，略过论文中后部分这里写一下总结。虽然是 2017 年的论文，但是这篇文章可谓是开辟了 AI 的新时代，它使用自注意力机制取代传统的 encoder
-decoder（编码器 - 解码器）中的递归层，且能够更快更好的训练，这是划时代的成就。但是当然也有缺点，attention 机制的计算成本增长很快，就像是更贵更 “聪明” 的 GPT 版本响应速度其实会更慢一些一样。
最后收藏了一下 https://github.com/tensorflow/tensor2tensor。
这篇论文很值得我们读。
